# Этапы выполнения задания

## Этап 1. Сбор данных

Выберите как минимум два источника данных и получите к ним доступ:

- API VirusTotal — для проверки IP, файлов или доменов;
- API Vulners — для поиска уязвимостей;
- логи Suricata — события сетевой безопасности.

При необходимости создайте папку с логами или подготовьте примеры данных.

## Этап 2. Анализ данных

Напишите скрипт, который:

- выполняет запросы к API или загружает логи;
- анализирует полученные данные: ищет опасные уязвимости (например, с высоким CVSS-баллом), определяет подозрительные IP или трафик (например, частые DNS-запросы).

Для анализа используйте библиотеки pandas, json, requests.

## Этап 3. Реагирование на угрозы

Добавьте логику реагирования. Если найдена угроза:

- выведите сообщение о найденной угрозе или сымитируйте блокировку IP;
- отправьте уведомления в Telegram или на email.

Реальное взаимодействие с системами блокировки не требуется, достаточно имитации — вывода сообщения в консоль.

## Этап 4. Формирование отчёта и визуализация

Сохраните результаты анализа в формате .json или .csv.  
Постройте один график по результатам анализа, например, топ-5 IP или распределение CVSS-баллов. Сохраните график в файл .png.
